Detailed Description

The idea of a "variant queue" here is to encode multiple frame types, with attached dynamically-sized payloads and transmit them via MemoryQueue.

Encoding looks like this: uint8 | N bytes | M bytes type id (derived from TypeList) | frame/header | payload

In this file you can find few helper structures and functions for operating on a queue with this encoding.

Note - structs used as headers need to be PODs. Unless unaligned read/write is acceptable on the target platform, they should also have 1-byte alignment requirement.

See the io module documentation for more info and examples.
