Detailed Description

Interface for writing slices of bytes with variable size to a data channel.

Member Function Documentation

◆ allocate()

virtual ::etl::span<uint8_t> io::IWriter::allocate ( size_t  size)
pure virtual

Allocates a slice of a given number of bytes.

Returns
- Slice of size bytes if underlying queue was not full and size <= maxSize()
  • Empty slice otherwise

Calling allocate multiple times before calling commit will re-allocate, i.e. a new slice of bytes will be returned and the previous allocation is lost and will not be automatically committed. This can also be used to trim the allocated slice to the right size before committing, thus calling allocate will NOT modify the memory (e.g. zero it).

Implemented in io::SplitWriter< N >, io::MemoryQueueWriter< Queue >, and io::BufferedWriter.

◆ commit()

virtual void io::IWriter::commit ( )
pure virtual

Makes the previously allocated slice available for the reader.

Calling commit() makes a new allocation mandatory! The previously allocated slice becomes invalid and must not be used anymore. Calling commit() without a previous successful allocation has no effect.

Implemented in io::SplitWriter< N >, io::MemoryQueueWriter< Queue >, and io::BufferedWriter.

◆ flush()

virtual void io::IWriter::flush ( )
pure virtual

Flushes any buffered data which has not yet been sent.

flush() is part of the interface to make subclassing, that implements buffering to e.g. improve bandwidth usage possible. It might have an empty implementation.

Implemented in io::SplitWriter< N >, io::MemoryQueueWriter< Queue >, and io::BufferedWriter.
